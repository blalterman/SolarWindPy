solarwindpy.plans.issues_from_plans moduleÔÉÅ

Create GitHub issues from plan files.

The script parses markdown files under solarwindpy/plans that contain YAML frontmatter and converts them into GitHub issues. Frontmatter fields name, about and labels are used for the issue metadata while the markdown content beginning with ## üß† Context becomes the issue body. The final issue title combines the containing directory name with any numeric prefix in the filename to preserve ordering. Optionally a specific subdirectory can be scanned via the -d/--directory CLI argument. Authentication uses a token supplied via --token or the GITHUB_TOKEN_SOLARWINDPY.
